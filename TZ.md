# Техническое задание: AI-ассистент юридического отдела

## 1. Общие сведения

**Название проекта:** Lawer — AI-ассистент юридического отдела
**Целевая аудитория:** Юристы компании (штатный юридический отдел)
**Юрисдикция:** Российская Федерация
**Цель:** Автоматизация рутинных задач юристов, сокращение времени на типовые операции, повышение качества и единообразия юридических документов.

---

## 2. Функциональные требования

### 2.1. Чат-интерфейс (основной способ взаимодействия)

Пользователь общается с ассистентом через чат-интерфейс (аналогично ChatGPT/DeepSeek):

- Ввод запроса на естественном языке (русский)
- Стриминг ответов (SSE/WebSocket) с плавным появлением текста
- Рендеринг Markdown в ответах (таблицы, списки, выделение)
- История диалогов с возможностью возврата к предыдущим
- Загрузка файлов (документы .docx, .pdf, .txt, .rtf) для анализа
- Скачивание сгенерированных документов

### 2.2. Анализ документов

- Загрузка юридического документа и получение краткого резюме
- Выявление ключевых условий, сроков, обязательств сторон
- Определение рисков и потенциально невыгодных условий
- Сравнение двух версий документа (diff)
- Проверка документа на соответствие внутренним стандартам компании

### 2.3. Генерация документов

- Создание типовых договоров по шаблонам с заполнением параметров:
  - Договор поставки
  - Договор оказания услуг
  - Договор аренды
  - Договор подряда
  - NDA (соглашение о неразглашении)
  - Трудовой договор
  - Дополнительные соглашения
- Генерация претензий и досудебных уведомлений
- Генерация доверенностей
- Генерация приказов и внутренних распоряжений
- Экспорт в .docx с корректным форматированием

### 2.4. Правовая аналитика

- Ответы на юридические вопросы с ссылками на нормативные акты РФ
- Поиск по базе нормативных актов (ГК РФ, ТК РФ, НК РФ, ФЗ и т.д.)
- Анализ судебной практики по заданной теме
- Отслеживание изменений законодательства, релевантных для компании
- Подготовка правовых заключений (legal opinions)

### 2.5. Управление задачами юридического отдела

- Ведение реестра текущих задач и сроков
- Напоминания о дедлайнах (окончание сроков договоров, сроки исковой давности, сроки ответов на претензии)
- Календарь судебных заседаний и процессуальных сроков

### 2.6. База знаний

- Хранение шаблонов документов компании
- Внутренние регламенты и политики
- FAQ по часто задаваемым юридическим вопросам
- Возможность пополнения базы знаний юристами

## 3. Нефункциональные требования

### 3.1. Безопасность
- Аутентификация пользователей (логин/пароль, возможна интеграция с корпоративным LDAP/AD)
- Ролевая модель доступа (администратор, юрист, наблюдатель)
- Шифрование данных при передаче (TLS) и хранении
- Все данные хранятся на территории РФ (152-ФЗ)
- Логирование действий пользователей (аудит)
- Запрет передачи конфиденциальных данных во внешние LLM-сервисы без явного согласия

### 3.2. Производительность
- Время начала ответа (первый токен): < 2 секунды
- Поддержка до 50 одновременных пользователей
- Загрузка документов до 50 МБ

### 3.3. Доступность
- Работа в корпоративной сети (on-premise или private cloud)
- Веб-интерфейс, адаптированный под десктоп и планшет
- Поддержка браузеров: Chrome, Firefox, Edge (последние 2 версии)

### 3.4. Локализация
- Интерфейс полностью на русском языке
- Поддержка русской правовой терминологии

## 4. Архитектура системы

### 4.1. Общая схема

```
[Браузер] <--HTTPS--> [Frontend (Next.js)] <--API--> [Backend (Node.js/Python)]
                                                            |
                                                    +-------+-------+
                                                    |               |
                                              [LLM Engine]   [MCP Servers]
                                                    |               |
                                              +-----+-----+   +----+----+
                                              | Claude API |   | DB      |
                                              | или        |   | Docs    |
                                              | Local LLM  |   | Law DB  |
                                              +------------+   +---------+
```

### 4.2. Frontend
- **Стек:** Next.js, React, TypeScript, Tailwind CSS
- **Стейт-менеджмент:** Zustand
- **Чат-интерфейс:** кастомная реализация со стримингом (SSE)
- **Компоненты:**
  - Чат с историей диалогов
  - Панель загрузки/скачивания документов
  - Боковая панель навигации (диалоги, шаблоны, задачи, база знаний)
  - Административная панель (управление пользователями, шаблонами)

### 4.3. Backend
- **Стек:** Node.js + TypeScript (или Python + FastAPI — на выбор)
- **API:** REST + SSE для стриминга ответов LLM
- **База данных:** PostgreSQL
- **Хранение файлов:** MinIO (S3-совместимое хранилище) или локальная ФС
- **Очередь задач:** BullMQ (Redis) для фоновой обработки документов
- **ORM:** Prisma (Node.js) или SQLAlchemy (Python)

### 4.4. MCP-серверы (Model Context Protocol)

MCP-серверы обеспечивают LLM доступ к инструментам и данным:

| MCP-сервер | Назначение | Ключевые tools |
|---|---|---|
| `mcp-law-database` | Поиск по нормативным актам РФ | `search_law`, `get_article`, `get_law_changes` |
| `mcp-document-processor` | Работа с документами | `parse_document`, `generate_docx`, `compare_documents` |
| `mcp-template-engine` | Генерация документов по шаблонам | `list_templates`, `fill_template`, `create_template` |
| `mcp-task-manager` | Управление задачами и сроками | `create_task`, `list_tasks`, `set_reminder` |
| `mcp-knowledge-base` | Поиск по базе знаний компании | `search_kb`, `add_to_kb`, `list_faq` |
| `mcp-court-practice` | Анализ судебной практики | `search_cases`, `analyze_case`, `get_statistics` |

### 4.5. LLM-интеграция
- **Основная модель:** Claude API (Anthropic) или self-hosted LLM (при требованиях on-premise)
- **Промпт-инжиниринг:** системные промпты, специализированные на юридической тематике РФ
- **RAG (Retrieval-Augmented Generation):** для ответов с опорой на нормативную базу и внутренние документы
- **Векторная база:** pgvector (расширение PostgreSQL) для хранения эмбеддингов

## 6. Этапы разработки

### Фаза 1: MVP (4-6 недель)
- Базовый чат-интерфейс со стримингом ответов LLM
- Аутентификация пользователей
- Загрузка и анализ документов (резюме, ключевые условия)
- Базовая генерация документов (2-3 типа договоров)
- MCP-серверы: `mcp-document-processor`, `mcp-template-engine`
- Развертывание в Docker

### Фаза 2: Правовая аналитика (3-4 недели)
- Интеграция базы нормативных актов РФ
- RAG-поиск по законодательству
- MCP-серверы: `mcp-law-database`, `mcp-court-practice`
- Расширение набора шаблонов документов
- Сравнение документов (diff)

### Фаза 3: Управление задачами и база знаний (2-3 недели)
- Реестр задач с дедлайнами и напоминаниями
- База знаний с возможностью пополнения
- MCP-серверы: `mcp-task-manager`, `mcp-knowledge-base`
- Календарь сроков

### Фаза 4: Hardening (2-3 недели)
- Ролевая модель доступа
- Аудит-лог действий
- Оптимизация производительности
- Тестирование безопасности (security-gate)
- Документация пользователя

## 8. Ограничения и допущения

1. Ассистент не заменяет юриста — он помогает и ускоряет работу. Все критические решения принимает человек.
2. Ответы LLM могут содержать неточности — система должна предупреждать об этом пользователя.
3. При работе с конфиденциальными данными необходимо соблюдение 152-ФЗ о персональных данных.
4. Актуальность нормативной базы зависит от регулярности обновлений источника данных.
5. Для полноценной работы off-line (без внешних API) потребуется self-hosted LLM с достаточными вычислительными ресурсами.

---

## 9. Критерии приемки

- Юрист может вести диалог с ассистентом на русском языке
- Загруженный документ анализируется и выдается структурированное резюме
- Генерация договора по шаблону занимает < 30 секунд
- Ответы содержат ссылки на конкретные статьи нормативных актов
- Система доступна по HTTPS, аутентификация работает
- Все данные хранятся в PostgreSQL, файлы — в объектном хранилище
- Docker Compose поднимает весь стек одной командой
